# 06B — Transactional Email
> Elevique Client Portal · Resend Email Templates, Dispatch Helpers, and Error Resilience

This specification defines the transactional email messaging system used to coordinate onboarding and delivery operations for clients and staff.

---

## 1. Resend API Configuration

Transactional emails are dispatched using the **Resend API SDK** (`resend` NPM module) in server-side operations:
- **Sending Domain**: `hello@elevique.in` (Domain verified with SPF, DKIM, and DMARC records).
- **Environment Key**: `RESEND_API_KEY` validated on project startup.
- **Limits**: Configured to stay within standard API caps (e.g. 3,000 emails/month on free tier).

---

## 2. Notification Templates Matrix

Two core templates are defined inside `src/lib/email/templates.ts`:

| Template Helper | Recipient | Trigger Event | Key Context Injected |
|---|---|---|---|
| `welcomeEmail` | New Client or Team Member | Admin finishes account creation in `/admin/users/new`. | User Full Name, Login email, autogenerated password, role-specific login URL |
| `milestoneCompleteEmail` | Client owner of the Project | Team or Admin marks a milestone phase status as `'completed'`. | Client name, Project name, Completed Milestone Phase (e.g. Scripting), Project Dashboard Link |

- **Branded Styling**: Rendered using clean, inline CSS styling matching the studio aesthetics (rich dark accents, outfit typography, centered content grids).
- **Fallbacks**: Renders a plain-text payload alongside the HTML version for older mail clients.

---

## 3. Resilience & Error-Handling Patterns

To prevent mail dispatch errors from interrupting database writes, the system isolates email operations:
- **Non-Blocking Execution**: Email trigger functions run asynchronously. They are not awaited at points that block user interface updates, or they are placed inside try-catch scopes that log failures to Sentry/console rather than rolling back DB transactions.
- **Example Pattern**:
  ```ts
  try {
    const resend = new Resend(process.env.RESEND_API_KEY)
    await resend.emails.send({ ...payload })
  } catch (error) {
    console.error('Failed to send onboarding email:', error)
    // Sentry capture exception
  }
  ```
- **Logging**: Captures status codes and API responses inside the server runtime log for tracking delivery status.
