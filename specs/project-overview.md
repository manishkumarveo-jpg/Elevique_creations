# Project Overview
> Elevique Client Portal · Custom Client Portal and Collaborative Dashboard System

This document outlines the core vision, objectives, and success criteria of the Elevique Creative AI Visuals Studio client portal.

---

## 1. Project Mission & Objectives

Elevique is a creative AI visuals studio that designs commercials, CGI/3D visual renders, and brand social campaigns. To scale client interactions and replace manually maintained Notion workspace links, Elevique is launching a custom-branded client collaboration portal.

### Core Problems Solved

- **No Authentication Security**: Notion links can be accessed by anyone possessing the URL. Elevique secures projects behind individual user accounts.
- **Data Isolation Rules**: Notion links do not support role boundaries; clients can browse background details. Elevique forces strict multi-role views.
- **Manual Overhead**: Setting up project pages, checklist requirements, and directories is currently done manually. Elevique automates project workspace generation.
- **File Access Protections**: External drafts are protected from unauthorized downloads or leakage by managing uploader link verification.

---

## 2. Multi-Portal Routing Matrix

Users access the system through three role-specific consoles, guarded by Edge Middleware:

```
┌─────────────────┐
│  Public Access  ├─► Login Pages: `/admin/login`, `/team/login`, `/portal/login`
└────────┬────────┘
         │
         ├──► Role = 'admin'        ──► `/admin/*`  (System settings, users, projects)
         ├──► Role = 'team_member'  ──► `/team/*`   (Work on assigned projects only)
         └──► Role = 'client'       ──► `/portal/*` (View milestones, feedback, deliverables)
```

---

## 3. Key Business Rules

1. **No Self-Registration**: Administrators create all accounts manually. Users receive autogenerated credentials via welcome emails.
2. **Strict Project Isolation**: Client accounts are completely isolated. Database RLS guarantees Client A cannot browse or query projects belonging to Client B.
3. **Team Scoping**: Creative staff (team members) only see projects where they are explicitly registered as workers in the assignments junction table.
4. **Link-Based Assets**: To reduce hosting overhead and storage costs, files are saved as third-party URLs (Google Drive, Frame.io) that open in new tabs.
5. **Immutable Audit Logs**: Key actions across user management, milestone completions, files, and deliverables are recorded to a read-only log database.
